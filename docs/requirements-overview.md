# 需求理解与模块拆解

## 核心目标
设计并实现一个面向滑坡监测业务的智能识别系统，支持多角色协作、图像识别、灾情评估和预警管理。

## 角色与权限
- 超级管理员：全局配置、账号管理、权限分配、系统审计
- 管理员：监测点管理、识别任务管理、预警策略管理
- 监测人员：上传图像、查看识别结果、接收预警

## 功能模块
1. 用户登录与鉴权
2. 角色权限分级（RBAC）
3. 滑坡图像上传与 AI 识别
4. 灾情等级自动判定（轻度/中度/重度）
5. 预警推送（按监测点和等级触发）
6. 后台管理（用户、任务、预警、日志）

## 非功能需求
- 安全性：JWT 鉴权、接口权限控制、敏感操作审计
- 可扩展性：识别模型可替换、预警通道可扩展
- 可维护性：前后端分层清晰、统一返回规范
