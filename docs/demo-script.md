# 答辩演示脚本（建议 5-8 分钟）

## 0. 开场（20 秒）
- 项目名称：基于人工智能算法的滑坡识别系统
- 目标：实现滑坡识别、灾情分级、实时预警和管理闭环

## 1. 系统启动（30 秒）
- 展示一键启动命令：
  - `powershell -ExecutionPolicy Bypass -File .\start-all.ps1`
- 说明前后端自动启动，降低部署复杂度

## 2. 登录与角色（40 秒）
- 用 `admin` 登录
- 展示左侧菜单随角色变化（管理员可见用户管理/灾情动态）

## 3. 识别任务（90 秒）
- 上传一张样例图
- 点击“开始识别”
- 展示：
  - 可视化框
  - 灾情等级（I/II/III/IV）
  - 面积、最大置信度、是否触发预警

## 4. 历史记录（60 秒）
- 打开“历史记录”
- 按时间和等级筛选
- 演示 CSV 导出

## 5. 实时预警（90 秒）
- 打开“灾情动态”
- 再触发一次达到 II 级及以上的识别
- 展示：
  - WebSocket 实时新增预警
  - 未确认数量变化
  - 点击“确认”后状态从未确认 -> 已确认

## 6. 个人中心（40 秒）
- 展示个人信息（用户名、中文角色、手机号）
- 点击“修改资料” -> 保存姓名/手机号

## 7. 收尾（30 秒）
- 总结闭环：识别 -> 分级 -> 预警 -> 处置确认 -> 历史追溯
- 强调工程价值：权限控制、实时通信、可维护部署

---

## 备选 Q&A 口径
- **Q: 为什么用 WebSocket？**
  - 预警要求低延迟推送，轮询会增加延迟和服务器压力。
- **Q: 为什么要预警确认？**
  - 把“发现”升级为“处置闭环”，可追溯、可统计。
- **Q: 阈值如何调优？**
  - 当前通过配置项控制，可根据误报/漏报在不同场景下微调。
